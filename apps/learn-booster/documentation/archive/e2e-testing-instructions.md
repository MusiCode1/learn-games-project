# הוראות להרצת בדיקות קצה לקצה

בדיקות הקצה לקצה משתמשות ב-Playwright כדי לדמות משתמש אמיתי שמתחבר לאתר ג'ינג'ים, מזריק את הקוד של gingim-booster, ובודק את הפונקציונליות.

## 1. התקנת Playwright

אם עדיין לא התקנת את Playwright, הרץ את הפקודות הבאות:

```bash
# התקנת חבילת Playwright
npm install --save-dev @playwright/test

# התקנת הדפדפנים הדרושים
npx playwright install chromium
```

## 2. וידוא שקובץ .env מכיל את פרטי ההתחברות

וודא שקובץ `.env` בתיקיית הפרויקט מכיל את פרטי ההתחברות לאתר ג'ינג'ים:

```
GINGIM_USER=your_username
GINGIM_PASS=your_password
```

## 3. בניית הפרויקט

וודא שהפרויקט בנוי ומוכן:

```bash
npm run build
```

## 4. הרצת הבדיקה

הרץ את הבדיקה באמצעות הפקודה:

```bash
npx playwright test
```

הבדיקה תפתח דפדפן, תתחבר לאתר ג'ינג'ים, תזריק את הקוד של gingim-booster, תפתח משחק ותבדוק את הפונקציונליות.

## 5. צפייה בתוצאות

לאחר סיום הבדיקה, תוכל לראות את התוצאות בקונסולה. בנוסף, ייווצר צילום מסך בנתיב `tests/e2e/gingim-test-result.png` שמראה את המצב הסופי של הבדיקה.

אם הבדיקה נכשלה, תוכל לראות דוח מפורט בנתיב `playwright-report/index.html`:

```bash
npx playwright show-report
```

## הערות

- הבדיקה מנסה לדמות משחק ב"סדר את החדר" על ידי לחיצה על מקומות שונים בקנבס. ייתכן שתצטרך להתאים את הלוגיקה הזו בהתאם למשחק הספציפי שאתה רוצה לבדוק.
- אם אתה רוצה לבדוק משחק אחר, שנה את שורה 121 בקובץ `tests/e2e/gingim-test.js` לשם המשחק הרצוי.
- הבדיקה מוגדרת לרוץ במצב גלוי (לא headless) כדי שתוכל לראות מה קורה. אם אתה רוצה להריץ במצב headless, שנה את ההגדרה `headless: false` ל-`headless: true` בקובץ `playwright.config.js`.
